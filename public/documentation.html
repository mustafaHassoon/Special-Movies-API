Special Movies API

Essential Features: ● Return a list of ALL movies to the user. ● Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user. ● Return data about a genre (description) by name/title (e.g., “Thriller”). ● Return data about a director (bio, birth year, death year) by name. ● Allow new users to register. ● Allow users to update their user info (username, password, email, date of birth). ● Allow users to add a movie to their list of favorites. ● Allow users to remove a movie from their list of favorites. ● Allow existing users to deregister.

Business logic URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of all movies /movies GET None A JSON object holding data about all the movies
Get details about a single movie by title /movies/[title] GET None A JSON object holding data about a single movie, containing a title, description, genre property, director property and image. Example:

{ title : 'Iron Man', description : 'After being held captive in an Afghan cave, billionaire engineer \ Tony Stark creates a unique weaponized suit of armor to fight evil.', genre : 'Action', director : 'Jon Favreau', image : 'iron_man.png' }
Get data about a genre of a movie by a title /genres/[name] GET None A JSON object holding data about a movie genre. Example:

{ name : 'Comedy', description : 'A comedy film is a genre of film in which the main emphasis is on humour. These films are designed to make the audience laugh through amusement and most often work by exaggerating characteristics for humorous effect.' }

Get a data about director by name /directors/[name] GET None A JSON object holding data about a director. Example:

{ name : 'Greg Mottola', born : 'July 11, 1964' }

Add new user /users POST A JSON object holding data about the user to add, structured like:

{ username : 'janedoe123', password : 'password123', email : 'example@gmail.com', date_of_birth : "January 1, 1990" }

A JSON object holding data about the user that was added, including an ID:

{ id : 1, username : 'janedoe123', password : 'password123', email : 'example@gmail.com', date_of_birth : "January 1, 1990" }

Update a user info /users/[username] PUT A JSON object holding data about the user to add, structured like:

{ username : 'janedoe123', password : 'password123', email : 'example@gmail.com', date_of_birth : "January 1, 1990" }

A text message indicating the user's information was updated
Allow users to add favorite movies /users/[username]/favorites POST None A message indicating the movie was added to the user's list of favorites
Allow user to remove a movie from their favorite list of movies /users/[username]/favorites DELETE None A message indicating the movie was removed from the user's list of favorites
Allow existing users to deregister /users/[username] DELETE None A message indicating the user's account was deleted
